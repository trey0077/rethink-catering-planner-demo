# Toast Restaurant BEO Expanded Notes

Source: `Toast Restaurant BEO Expanded.pdf` provided by Trey.

## Useful extraction

A Banquet Event Order (BEO) is described as both a contract and an operational guide for a catering/restaurant event. It helps align the restaurant/caterer staff and the event organizer.

Core BEO fields in the template:

- Prepared by
- Customer name
- Event date
- Contact
- Menu
  - Food
  - Beverage
  - Other
- Event details
  - event name
  - group sizes
  - space
  - occasion
  - style of service
- Event timeline
  - event start
  - time blocks
  - end time
- Additional notes
- Amount due
- Deposit
- Customer signature/date
- Employee signature/date

Toast Catering & Events positioning also emphasizes:

- customizable event/order templates
- online lead forms
- shared calendar for upcoming events/status
- deposits applied directly to orders
- lead management, prep tools, invoicing

## Product implications for this demo

The public planner should not merely recommend food. It should create the front half of a BEO-ready handoff:

- event date and availability note
- group size / headcount
- service style
- beverage/alcohol expectation
- rental/support needs
- amount-range planning
- hidden fees and deposit prompts
- copy/paste consultation summary that can become intake/BEO notes

This supports Trey's point: a caterer can use its own menu/event data to help the client make a good decision and provide ~80% of what the caterer needs before the first call.
